# SMB Enumeration Analysis

## Detection ID

`DET-005`

## Attack Type

SMB Enumeration / Service Discovery

## Objective

Identify SMB discovery traffic generated by Nmap NSE scripts against ports `139` and `445`.

## Lab Context

| Role | IP Address |
|---|---:|
| Attacker | `192.168.1.4` |
| Victim | `192.168.1.3` |
| Target Ports | `139/TCP`, `445/TCP` |

## Command Used

```bash
sudo nmap -sS -p 139,445 --script smb-os-discovery 192.168.1.3
```

## Wireshark Display Filter

```text
tcp.port == 139 || tcp.port == 445
```

## Packet-Level Evidence

Expected evidence:

| Evidence | Description |
|---|---|
| Ports | `139` and `445` |
| Protocol | SMB / NetBIOS / TCP |
| Session behavior | Short request/response burst |
| SMB messages | Session Setup, negotiation, possible logoff |
| Data transfer | No long file-transfer session |
| Intent | Service discovery and system information gathering |

## Why This Indicates SMB Enumeration

SMB enumeration is usually short, targeted, and query-focused. A normal SMB file transfer would involve longer sessions and more sustained data transfer. In this lab, the traffic pattern is consistent with a scanning tool establishing SMB sessions to query host or service information.

## Investigation Notes

SMB enumeration is a discovery activity. Attackers often enumerate SMB before attempting credential attacks, share access, lateral movement, or exploitation.

## Defensive Recommendations

- Disable SMB if not required.
- Block ports `139` and `445` from untrusted networks.
- Disable anonymous enumeration.
- Restrict SMB shares using least privilege.
- Monitor for SMB scanning and session setup bursts.
- Segment systems that require SMB access.
- Alert on Nmap-like SMB script activity.
